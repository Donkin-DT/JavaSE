######面向对象 与 面向过程

```java
面向对象和面向过程都属于解决问题的思考方式

面向过程：以执行者角度思考问题，更侧重于“怎么做”，也就是做的行为，更适合解决较小的程序，
面向对象：以指挥者角度思考问题，更侧重于“谁来做”，也就是做的对象，更适合解决较大的程序

面向过程和面向对象并不是完全分离的，二者相互依赖。一般来讲，宏观上用面向对象思考，微观上用面向过程
案例1：
	开了小菜馆：
    买菜
    洗菜
    切菜
    做菜
    上菜
    结账

    采购员：买菜
    后厨：洗菜、切菜
    厨师：做菜
    服务员：上菜
    收银：结账
案例2：
	一个人单身：
	收拾屋子
	洗衣服
	做饭
	学习
	看电影	
	玩游戏
	
	有了女朋友：
	女朋友：收拾屋子

	女朋友：做饭
	你：学习
	你：看电影	
	你：玩游戏
	
总结：面向对象的使用步骤：😀
    1、分析程序中涉及到了哪些对象
    2、根据对象的特征抽取出 类
    3、按步骤，调用对象或类中的功能
```
######类 与 对象

```java
对象：Java中万事万物皆对象（反射）😊
	定义：
		客观世界中存在的一切可以被描述的事物，都称为对象
	
如何区分开每一个对象？
	通过对象的特征（属性和方法）
属性：
	对象的外观特征
	比如：姓名、年龄、颜色、价格等
方法：
	对象的行为特征或功能
	比如：能装水、吃、喝、能计算
	
类
	定义：具有相同属性和方法的一组对象集合的抽象，相当于程序员经过归纳总结出来的产物

关系：
	类是由对象总结出来的，属于对象的类型
	对象是通过类创建出来的具体的实例
	
区别：
	类是抽象的，不能实际操作的
	对象是具体的，客观存在
```

######类的成员（属性）

```java

语法：
	同变量的使用语法
	数据类型 属性名;//[通用写法]

	根据需要也可以手动赋值：
	数据类型 属性名 = 值;
注意：
	数据类型可以为任意数据类型，包含基本类型或引用类型

特点:
	1.属性可以不赋值就直接使用,因为有默认值😀
		int——>0 
		double——>0.0
		char——>\u0000
		boolean——>false
		引用类型——>null
        
    2、属性伴随着对象存放在堆中，不同对象的属性是独立的，互不影响！
    
    3、属性也属于变量，按作用域不同进行分类：😀
		局部变量：定义在方法中或某个代码块中，作用域较小，仅仅是定义它的方法或代码块内！
		全局变量：定义在类体中，属于类的成员（属性），作用域较大，是整个类！又称为成员变量或属性

局部变量与全局变量区别：😀
	①作用域不同
	②存储位置不同
		局部变量存储在栈中
		全局变量存储在堆中
	
	③生命周期不同
		局部变量伴随着方法或代码块的执行而创建，伴随着方法或代码块的结束而消亡
		全局变量伴随着对象的创建而创建，伴随着对象的消亡而消亡
	
	④默认值的问题
		局部变量没有默认值
		全局变量有默认值
	
	⑤重名的问题
		两个局部变量如果作用域有交集，则不可以重名，否则报错！
		一个局部变量和一个全局变量，可以重名！访问时遵循就近原则！
	
	⑥访问修饰符的问题
		局部变量不可以使用访问修饰符
		全局变量可以使用访问修饰符
    
```
######类的成员(方法) 😀

```java
概念：又称为函数，用于描述类或对象的行为特征或功能
	比如：吃、学习、求和、求最值、查找、排序等

注意：方法属于类的成员，需要定义在类体内！😀

好处:
	1.提高代码的重用性和维护性
	2.方法隐藏了实现细节，从调用层面上讲，调用比较方便，因为只需要关心两件事
		①叫什么（方法名）
		②干什么（方法功能）

定义语法:
	[修饰符] 返回类型  方法名(参数列表){
		方法体（实现细节）
	}

五要素：😀
	要素一：修饰符
	特点：可选
	public 公共的，任意类都可以调用该方法
	
	要素二：返回类型
	说明：某些行为或功能执行结束后需要返回给调用方一个具体的值，则将值的类型标注成返回类型，如果行为或功能执行结束后没有返回值，则返回类型写成void即可
	特点：
		①一个方法至多有一个返回值，要求返回值的类型和返回类型一致或兼容
    	②如果方法有返回值，则方法体中必须有“return 值;”语句
     	如果方法没有返回值，则方法体中可以不用有return语句，返回类型为void，如果非要有return语句，可以使用“return”;
		③返回类型支持任意数据类型，包含基本类型或引用类型
	
	要素三：方法名
	遵循标识符的命名规则和规范（同变量的规范）：见名知义，且遵循驼峰命名法 
	
	要素四：参数列表
	说明：某些方法的执行需要调用方传递源数据，否则无法执行。我们把这些源数据称为参数
	特点：
		①参数可以为多个，语法如下：
		（类型 参数名1,类型 参数名1）
		②参数类型可以为任意类型，包含基本类型或引用类型
		③参数的本质就是一个局部变量，也必须赋值后才能使用，但这个赋值一般是隐式赋值的（传参时）
		④方法定义时的参数称为 形式参数，简称形参；方法调用时的参数称为 实际参数，简称实参
形参和实参的类型、个数必须一致，名称无要求

	要素五：方法体
	可以包含0条或任意多条逻辑语句。
	形式：输入、输出、变量、运算、分支、循环、方法调用

方法分类：
	1.无返回无参数
	2.有返回无参数
	3.无返回有参数
	4.有返回有参数
```
######方法调用😀
```java
理解：
	方法的调用，其实就是执行方法体中的每一句话

语法：
	1、同一个类内的方法调用
		直接调用即可。示例：方法名();
	2、跨类的方法调用
		通过对象或类名调用。示例：对象.方法名();
	注意：
		①如果方法有返回值，则调用时最好接受或处理该返回值
		②如果方法有参数，则要求调用时必须传入对应类型和个数的参数
		
调用机制：😀
	每次方法调用，都会为该方法分配一个栈帧（一块空间），也就是“入栈”操作
	每次方法调用结束，都会释放对应的栈帧，也就是“出栈”操作
	每个不同的方法的栈帧是独立的，互不影响
	栈的结构：“先进后出”

方法的传参😀
	理解：当调用方法时，实参将值传递给形参的过程（相当于参数的赋值），称为方法的传参
	特点【面试题】：
		java属于值传递，基本类型的值就是元素内容；引用类型的值是地址
		参数类型为基本类型，传递的是“元素内容”，形参的改变不影响实参
		参数类型为引用类型，传递的是“地址”，形参的改变影响实参
	注意：如果形参在方法体中更改了新地址，则它的改变不影响实参！
```

######方法重载😀
```java
概念：
	java中允许同一个类中多个同名方法的存在，但要求参数列表必须不同。这种现象称为方法重载（OverLoad）
	经典的方法重载案例：
	System.out.println();
	
好处：
	1、解决了起名太麻烦的问题
	2、解决了记名太麻烦的问题

要求：😀
	①方法名必须相同
	②参数类型、个数、顺序至少有一样不同。参数名无要求
	③返回类型无要求
	④修饰符无要求
```

######可变参数方法😀

```java
概念：
	jdk5.0新特性，将一个类中多个同名、同返回、同参数类型，仅仅参数个数不同的方法抽取成一个方法，这个方法称为可变参数的方法
	
好处：
	提高代码的重用性
	
语法：😀
	修饰符 返回类型 方法名(参数类型...参数名){ }
	注意：使用可变参数时，当做数组使用
	经典案例：
    	String.format()
		System.out.printf()
        
特点：😀
	①可变参数的实参支持0个或任意多个参数
	②当调用的方法有多个重载，注意：调用顺序是先找最一致，然后找兼容，最后找可变的
	③可变参数是否可以和普通类型的参数同时出现在形参列表？
		可以！但要求可变参数必须放在最后
	④一个形参列表中可以有多个可变参数吗？
		不可以！
	⑤可变参数方法是否可以和同一类型的数组参数方法构成重载
		不可以！
	⑥可变参数的实参支持数组吗？
		可以！
```

###### 包

```java
好处:
	1、分类管理文件，方便管理和查找
	2、解决了同一个项目中同类名问题

创建：
	1、右击src——新建package——然后新建class
	2、右击src——新建class的同时指定package
	3、在类的最上方添加package语句
		package 包名;

包的命名：
	1.命名规则
		标识符+小圆点  
		示例：demo.study
		demo.1study  ×
		java.demo  ×
		
	2.命名规范
		纯小写字母，且遵循域名倒置的写法
		示例：com.atguigu
		www.atguigu.com
		通用的写法：com.公司名.部门名.项目名.业务模块名;😀
		com.atguigu.service
		com.atguigu.view
		
使用特点：
	1、package语句
		①如果某类在指定的包下，则要求该类上方必须添加包的声明语句.
			语法：package 包名;
		②数量：对于一个类来讲，至多有一个包的声明语句
		③位置：放在class的最上方
		
	2、import语句
		①如果访问其他包中的类，则必须导入该类所在的包
			语法：import 包名.类名;
				或import 包名.*;
		②数量：对于一个类来讲，可以有多条导包语句，没有顺序要求
		③位置：package——import——class
		④注意：😀
			java.lang包是系统默认导入的，不用手动导包
			如果多个包中有同名类，都需要导入到本类中，不能添加多条导包语句，需要设置全类名
			demo2.Student s = new demo2.Student();

```

######封装😀

```java
好处：
	1、隐藏了内部实现细节
	2、提高数据的安全性
	3、对于属性的封装来讲，避免了调用方的任意赋值的危险，提高了属性的安全性！

分类：
	广义的封装：方法、类、包
	狭义的封装：属性的封装  😀

体现：😀
	① 将属性私有化
	② 提供公共的set方法，用于对属性更新（里面可以对赋值的数据进行有效性验证）
	③ 提供公共的get方法，用于访问属性的值
	注意：	如果希望属性是只读的，则只需要提供get方法；如果希望属性是只写的，则只需要提供set方法
	
规范:
	将所有的属性进行合理封装。
```

###### 构造器😀

```java
理解：
	构造器又称为构造方法，属于类中的第三大成员。同方法类似，也有方法体，也需要调用才能执行，但没有返回类型，并且不是通过对象或类显式调用
	
语法：
	【修饰符】	方法名(参数列表){
  		构造器体
  	}
  	
  	注意：😀
	  1、没有返回类型
	  2、方法名必须和当前的类名一致！
	  3、其他同方法！

好处：
	1、用于创建对象
 	2、通过构造器，尤其是有参构造器，做一些初始化的操作，从一定程度上提高代码的重用性！
 
特点：😀
	1、类中如果没有显式的声明构造器，系统将默认提供一个无参的构造器。
	（每个类都有构造器）
  	2、类中显式的声明了构造器，系统将不再提供构造器
  	3、构造器可以重载
  	4、构造器，对于一个对象来讲，仅仅调用一次
  	5、构造器之间也可以互相调用,要求必须通过this调用
  		语法：this(实参列表);
  		注意：this调用构造器 只能放在构造器的第一句，而且只能有一个！
```
